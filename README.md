# 私人本地笔记系统（Notes-Web V2.0）

一个运行在 Windows 本地环境的轻量级网页笔记工具，面向单用户使用，强调 **像 VS Code 一样的单文档编辑体验**、本地数据持久化，以及可管理的历史版本能力。

## 产品简介

Notes-Web V2.0 是一次视觉体验全面升级。

V1 解决的是"能记录、能保存、能看历史"的问题；
V1.1 进一步解决的是"**编辑体验更像真正写代码/写文档工具**"的问题；
V1.2 重点解决的是"**数据写入更稳、预览更顺、交互更安全**"的问题；
V2.0 重点解决的是"**视觉体验更炫、界面更具赛博朋克风格**"的问题。

因此，这一版重点做了：

- 全新赛博朋克视觉风格，品红 + 电光蓝主色调
- 霓虹发光效果，边框、按钮、标题带有适度发光
- 扫描线动画，背景有缓慢移动的扫描线效果
- 故障动画，标题带有轻微的故障动画效果
- CodeMirror 编辑器赛博主题适配
- 优化整体布局和视觉层次

## V2.0 当前能力

- 新建笔记
- 左侧笔记列表展示
- 左侧历史记录展示
- 单文档 Markdown 编辑
- 类似 VS Code 的编辑区布局
- Markdown 预览切换
- fenced code block 代码记录方式
- 手动保存到本地 JSON 文件
- 历史版本恢复为当前内容
- 删除单条历史版本
- 删除整篇笔记
- 原子写入与历史版本瘦身存储
- 请求期间关键按钮禁用
- Markdown 预览安全过滤
- 赛博朋克视觉风格

## 技术栈

- **后端**：Flask
- **前端**：HTML + CSS + JavaScript
- **编辑器**：CodeMirror（CDN 方式引入）
- **Markdown 渲染**：Marked（CDN 方式引入）
- **数据存储**：本地 JSON 文件

## 框架说明 / 系统架构

本项目仍然采用轻量、可维护的本地 Web 应用结构，不引入复杂的前端工程化工具链。

### 1. 整体架构

```text
浏览器页面
   ↓
Flask 本地服务
   ↓
本地 JSON 文件
```

### 2. 分层职责

#### 表现层（UI 层）

负责页面展示、交互与编辑体验：

- `templates/index.html`
- `static/css/styles.css`
- `static/js/app.js`

这一层完成：

- 赛博朋克风格的主界面布局
- 左侧笔记列表与历史记录面板
- 单文档编辑器交互
- Markdown 预览切换与渲染防抖
- 请求期间关键按钮禁用，避免重复操作
- Markdown 预览安全过滤
- 删除笔记 / 删除历史版本交互
- 调用后端 API 进行保存、恢复、删除

#### 接口层（应用层）

负责路由与 API 暴露：

- `app.py`

这一层完成：

- 首页渲染
- 笔记列表接口
- 创建笔记接口
- 获取笔记详情接口
- 保存笔记接口
- 恢复历史版本接口
- 删除笔记接口
- 删除历史版本接口

#### 存储层（数据层）

负责本地文件读写与数据结构维护：

- `note_store.py`
- `data/*.json`

这一层完成：

- 笔记 JSON 文件管理与原子写入
- 历史版本快照生成
- 历史版本恢复
- 历史版本删除
- 笔记删除
- 历史版本数据瘦身（持久化以 document 为主）
- 单文档内容与 blocks 结构之间的转换

### 3. 为什么这样设计

这套结构适合当前产品目标，原因有三点：

1. **简单**：不用引入 React、Vue、数据库、打包工具，直接可运行
2. **可维护**：前端、接口、存储职责清晰，后续容易定位问题
3. **可迭代**：以后如果要加入 SQLite、标签、搜索、导出，只需要在现有分层基础上扩展

## V2.0 编辑模型说明

当前界面仍然采用单文档编辑器；历史版本持久化以 `document` 为主，运行时再按需还原 `blocks`，以保证：

- 兼容已有数据文件
- 历史版本在数据瘦身后仍然可以恢复
- 后续仍然可以扩展更细粒度的内容能力

转换规则如下：

- 普通文本内容 → `text` block
- fenced code block → `code` block

例如：

```markdown
# Flask 学习记录

今天完成了本地笔记系统原型。

```python
print("hello")
```
```

保存后会被解析为文本块 + 代码块，并继续写入版本历史。

## 设计原则

本项目会持续遵守以下原则：

- **简单**：先把主链路做稳
- **可维护**：代码职责清晰，不做过度设计
- **可迭代**：为未来增强预留空间，但不提前堆功能
